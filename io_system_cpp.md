# Система введення-виведення в C++ — Контрольні запитання та відповіді

---

## 1. Основні положення системи введення-виведення C++

- Система введення-виведення C++ побудована на основі **ієрархії класів потоків** (`ios`, `istream`, `ostream`, `iostream` тощо), що знаходяться у заголовному файлі `<iostream>`.
- Вся робота з введенням-виведенням здійснюється через **потоки (streams)** — абстраговані канали передачі даних.
- Підтримуються **два рівні**: форматоване введення-виведення (через `>>`, `<<`) і неформатоване (через `get()`, `put()`, `read()`, `write()`).
- Система є **типобезпечною**: компілятор знає тип даних і обирає відповідну операцію автоматично.
- Підтримується **перевантаження операторів** `<<` і `>>` для користувацьких типів.
- Для файлового введення-виведення використовуються класи з `<fstream>`: `ifstream`, `ofstream`, `fstream`.

---

## 2. Що таке потік?

**Потік (stream)** — це абстракція, що представляє послідовний потік байтів між програмою та джерелом/приймачем даних (клавіатура, екран, файл, рядок у пам'яті).

- **Вхідний потік (input stream)** — дані надходять до програми.
- **Вихідний потік (output stream)** — дані виводяться з програми.
- **Двонаправлений потік** — підтримує обидва напрямки.

Потік буферизує дані та надає уніфікований інтерфейс незалежно від фізичного пристрою.

---

## 3. Потоки, що відкриваються під час запуску програми

| Потік | Об'єкт | Призначення |
|-------|--------|-------------|
| Стандартне введення | `cin` | Читання даних з клавіатури |
| Стандартне виведення | `cout` | Виведення даних на екран |
| Стандартні помилки (буферизований) | `cerr` | Виведення повідомлень про помилки (без буфера) |
| Стандартні помилки (небуферизований) | `clog` | Виведення повідомлень журналу (буферизований) |

```cpp
#include <iostream>
using namespace std;

int main() {
    int x;
    cin >> x;          // читання з клавіатури
    cout << x << endl; // виведення на екран
    cerr << "Помилка!" << endl; // виведення помилки
    clog << "Лог-запис" << endl;
}
```

## 4. Прапори формату

Прапори формату зберігаються у змінній типу `fmtflags` класу `ios` і керують форматом виведення.

### Основні прапори:

| Прапор | Призначення |
|--------|-------------|
| `ios::left` | Вирівнювання по лівому краю |
| `ios::right` | Вирівнювання по правому краю |
| `ios::internal` | Знак ліворуч, число праворуч |
| `ios::dec` | Десяткова система числення |
| `ios::oct` | Вісімкова система числення |
| `ios::hex` | Шістнадцяткова система числення |
| `ios::showbase` | Виводити префікс системи числення (`0`, `0x`) |
| `ios::showpoint` | Завжди виводити десяткову крапку |
| `ios::uppercase` | Великі літери для шістнадцяткових та `E` |
| `ios::showpos` | Виводити `+` для позитивних чисел |
| `ios::scientific` | Науковий формат (`1.23e+05`) |
| `ios::fixed` | Фіксована кількість знаків після крапки |
| `ios::boolalpha` | Виводити `true`/`false` замість `1`/`0` |
| `ios::skipws` | Пропускати пробіли при введенні |

### Приклади:

```cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    int n = 255;

    // Системи числення
    cout << dec << n << endl;      // 255
    cout << oct << n << endl;      // 377
    cout << hex << n << endl;      // ff
    cout << hex << uppercase << n << endl; // FF

    // Showbase
    cout << showbase << hex << n << endl;  // 0xff

    // Числа з плаваючою крапкою
    double pi = 3.14159265;
    cout << fixed << setprecision(3) << pi << endl;      // 3.142
    cout << scientific << setprecision(2) << pi << endl; // 3.14e+00

    // Вирівнювання
    cout << setw(10) << left  << "Ліво"  << "|" << endl; // "Ліво      |"
    cout << setw(10) << right << "Право" << "|" << endl; // "     Право|"

    // bool
    cout << boolalpha << true << " " << false << endl; // true false

    // showpos
    cout << showpos << 42 << endl; // +42
}
```

## 5. Функції, що визначають параметри формату

### Функції-члени класу `ios`:

| Функція | Призначення |
|---------|-------------|
| `setf(flags)` | Встановлює прапори формату |
| `unsetf(flags)` | Знімає прапори формату |
| `flags()` | Повертає поточні прапори |
| `width(n)` | Встановлює мінімальну ширину поля |
| `precision(n)` | Встановлює кількість значущих цифр |
| `fill(c)` | Встановлює символ заповнення |

### Маніпулятори з `<iomanip>`:

| Маніпулятор | Призначення |
|-------------|-------------|
| `setw(n)` | Ширина поля |
| `setprecision(n)` | Точність числа |
| `setfill(c)` | Символ заповнення |
| `setbase(n)` | Основа системи числення (8, 10, 16) |
| `setiosflags(f)` | Встановити прапори |
| `resetiosflags(f)` | Зняти прапори |

### Приклади:

```cpp
#include <iostream>
#include <iomanip>
using namespace std;

int main() {
    // width і fill
    cout.width(8);
    cout.fill('*');
    cout << 42 << endl;              // ******42

    // через маніпулятори
    cout << setw(8) << setfill('-') << 42 << endl; // ------42

    // precision
    cout.precision(5);
    cout << 3.14159265 << endl;      // 3.1416

    // setf
    cout.setf(ios::showpos | ios::fixed);
    cout.precision(2);
    cout << 3.14 << endl;            // +3.14
    cout.unsetf(ios::showpos);
    cout << 3.14 << endl;            // 3.14
}
```

## 6. Особливості файлового введення-виведення в C++

Для роботи з файлами підключають `<fstream>`:

- `ifstream` — читання з файлу.
- `ofstream` — запис у файл.
- `fstream` — читання і запис.

### Відкриття та закриття файлу:

```cpp
#include <fstream>
#include <iostream>
using namespace std;

int main() {
    // Запис у файл
    ofstream out("data.txt");
    if (!out) {
        cerr << "Помилка відкриття файлу!" << endl;
        return 1;
    }
    out << "Рядок 1" << endl;
    out << 42 << " " << 3.14 << endl;
    out.close();

    // Читання з файлу
    ifstream in("data.txt");
    string line;
    while (getline(in, line)) {
        cout << line << endl;
    }
    in.close();
}
```

### Режими відкриття файлу (`ios::openmode`):

| Прапор | Призначення |
|--------|-------------|
| `ios::in` | Відкрити для читання |
| `ios::out` | Відкрити для запису |
| `ios::app` | Дописувати в кінець |
| `ios::ate` | Перейти в кінець після відкриття |
| `ios::trunc` | Очистити файл при відкритті |
| `ios::binary` | Двійковий режим |

```cpp
fstream f("file.dat", ios::in | ios::out | ios::binary);
```

## 7. Оператори вставки та вилучення

- **Оператор вставки `<<`** (insertion operator) — виводить дані в потік.
- **Оператор вилучення `>>`** (extraction operator) — зчитує дані з потоку.

Обидва оператори **перевантажені** для всіх вбудованих типів і можуть бути перевантажені для користувацьких класів.

```cpp
int x = 10;
cout << x;   // вставка: виводить 10 у потік cout
cin  >> x;   // вилучення: зчитує значення зі потоку cin
```

Оператори повертають **посилання на потік**, що дозволяє ланцюгове використання:

```cpp
cout << "x = " << x << endl;
cin >> a >> b >> c;
```

## 8. Створення функцій вставки та вилучення для власних класів

Функції вставки/вилучення **не можуть бути членами** класу, для якого їх створюють, оскільки лівий операнд — це потік, а не об'єкт класу. Вони оголошуються як **дружні функції (friend)**.

### Мета:
Забезпечити зручний та природний синтаксис `cout << obj` і `cin >> obj` для власних типів.

---

## 9. Загальна форма функції вставки (`<<`)

```cpp
ostream& operator<<(ostream& потік, const ТипКласу& об'єкт) {
    потік << /* поля об'єкта */;
    return потік;
}
```

### Приклад:

```cpp
#include <iostream>
using namespace std;

class Point {
public:
    double x, y;
    Point(double x = 0, double y = 0) : x(x), y(y) {}

    friend ostream& operator<<(ostream& out, const Point& p);
};

ostream& operator<<(ostream& out, const Point& p) {
    out << "(" << p.x << ", " << p.y << ")";
    return out;
}

int main() {
    Point p(3.0, 4.5);
    cout << "Точка: " << p << endl; // Точка: (3, 4.5)
}
```

## 10. Загальна форма функції вилучення (`>>`)

```cpp
istream& operator>>(istream& потік, ТипКласу& об'єкт) {
    потік >> /* поля об'єкта */;
    return потік;
}
```

### Приклад:

```cpp
istream& operator>>(istream& in, Point& p) {
    in >> p.x >> p.y;
    return in;
}

int main() {
    Point p;
    cout << "Введіть x та y: ";
    cin >> p;
    cout << "Ви ввели: " << p << endl;
}
```

## 11. Чи можуть функції вставки та вилучення належати класу?

**Ні.** Функції `operator<<` і `operator>>` **не можуть бути членами** класу, для якого їх визначають, оскільки:

- Лівий операнд оператора `<<` — це об'єкт `ostream`, а не об'єкт класу.
- Якщо функція — член класу, лівим операндом завжди буде об'єкт цього класу.

Тому вони визначаються як **вільні (глобальні) функції**, зазвичай оголошені як `friend` для доступу до приватних полів.

```cpp
class MyClass {
    int value;
public:
    // friend — для доступу до private
    friend ostream& operator<<(ostream& out, const MyClass& obj);
    friend istream& operator>>(istream& in,  MyClass& obj);
};
```

## 12. Типи параметрів та тип значення, що повертається

| Функція | Перший параметр | Другий параметр | Повертає |
|---------|----------------|-----------------|----------|
| `operator<<` | `ostream&` | `const ТипКласу&` | `ostream&` |
| `operator>>` | `istream&` | `ТипКласу&` | `istream&` |

- Перший параметр — **посилання на потік** (не `const`, бо стан потоку змінюється).
- Другий параметр у `<<` — **`const` посилання** (об'єкт не змінюється при виведенні).
- Другий параметр у `>>` — **звичайне посилання** (об'єкт заповнюється зчитаними даними).
- Повертається **посилання на потік** для підтримки ланцюгових виразів.

## 13. Власні маніпулятори

**Маніпулятор** — це функція, що змінює стан потоку і може використовуватися безпосередньо в ланцюжку `<<` або `>>`.

### Мета:
Спрощення повторюваних налаштувань формату, покращення читабельності коду.

### Загальна форма маніпулятора без параметрів:

```cpp
// Для потоку виведення
ostream& ім'яМаніпулятора(ostream& потік) {
    // зміна стану потоку
    return потік;
}

// Для потоку введення
istream& ім'яМаніпулятора(istream& потік) {
    // зміна стану потоку
    return потік;
}
```

### Приклад:

```cpp
#include <iostream>
#include <iomanip>
using namespace std;

// Маніпулятор: встановити ширину 10 і заповнення '*'
ostream& wide_star(ostream& out) {
    out << setw(10) << setfill('*');
    return out;
}

// Маніпулятор: виводити числа у шістнадцятковому форматі з префіксом
ostream& hexfmt(ostream& out) {
    out << showbase << hex << uppercase;
    return out;
}

int main() {
    cout << wide_star << 42 << endl;  // ********42
    cout << hexfmt << 255 << endl;    // 0XFF
}
```

### Маніпулятор з параметром (через клас-помічник):

```cpp
struct SetWidth {
    int n;
    SetWidth(int n) : n(n) {}
};

ostream& operator<<(ostream& out, const SetWidth& sw) {
    out.width(sw.n);
    return out;
}

int main() {
    cout << SetWidth(12) << 99 << endl; //           99
}
```

## 14. Переваги введення-виведення C++ порівняно з мовою C

| Критерій | C (`printf`/`scanf`) | C++ (`cout`/`cin`) |
|----------|----------------------|---------------------|
| **Типобезпека** | ❌ Формат рядка не перевіряється компілятором | ✅ Тип визначається автоматично |
| **Розширюваність** | ❌ Не підтримує користувацькі типи | ✅ Перевантаження `<<` і `>>` для будь-якого класу |
| **Безпека** | ❌ Помилка у форматі → збій або UB | ✅ Компілятор виявляє невідповідності |
| **Ланцюгові виклики** | ❌ Неможливі | ✅ `cout << a << b << c` |
| **Маніпулятори** | ❌ Відсутні | ✅ Гнучке керування форматом |
| **Файловий ввід-вивід** | Ручне керування `FILE*` | ✅ Об'єктно-орієнтовані `fstream` |
| **Синтаксис** | `printf("%d %s", n, s)` | `cout << n << " " << s` — простіше |
| **Локалізація** | Обмежена | ✅ Підтримка `locale` |

### Ілюстрація типобезпеки:

```cpp
// C — помилка виявляється лише під час виконання
int x = 42;
printf("%s", x); // UB — жодної помилки компілятора!

// C++ — помилка виявляється під час компіляції
cout << x; // завжди правильно, тип відомий компілятору
```
