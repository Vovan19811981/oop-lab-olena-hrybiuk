# Виняткові ситуації в C++ — Контрольні запитання та відповіді

## 1. Що таке виняткова ситуація?

**Виняткова ситуація (exception)** — це подія, що виникає під час виконання програми і порушує нормальний хід її роботи. Як правило, це помилкова або нештатна ситуація: ділення на нуль, вихід за межі масиву, нестача пам'яті тощо.

C++ надає механізм структурованого оброблення таких ситуацій за допомогою операторів `try`, `catch` і `throw`.

---

## 2. Загальні форми операторів `try`, `catch` і `throw`

### `throw` — генерація виняткової ситуації

```cpp
throw вираз;
```

Генерує (кидає) виняток. `вираз` може бути будь-якого типу: ціле число, рядок, об'єкт класу тощо.

```cpp
throw 42;
throw "Помилка!";
throw std::runtime_error("Щось пішло не так");
```

### `try` — блок, що охороняється

```cpp
try {
    // код, який може спричинити виняткову ситуацію
}
```

Визначає блок коду, всередині якого відстежуються виняткові ситуації.

### `catch` — оброблення виняткової ситуації

```cpp
catch (тип параметр) {
    // код обробника
}
```

Перехоплює виняток певного типу та виконує відповідний код-обробник.

### Повна форма:

```cpp
try {
    // код, що може кинути виняток
}
catch (тип1 змінна1) {
    // обробник для типу1
}
catch (тип2 змінна2) {
    // обробник для типу2
}
// ...
catch (...) {
    // обробник для будь-якого типу
}
```

---

## 3. Спільна робота `try`, `catch` і `throw`

1. Виконання входить у блок `try`.
2. Якщо всередині блоку `try` (або у функції, що викликається з нього) виконується `throw`, генерується виняток.
3. C++ **припиняє** нормальне виконання блоку `try` і **шукає** відповідний блок `catch` за типом кинутого значення.
4. Якщо відповідний `catch` знайдено — виконується його тіло.
5. Після завершення `catch` виконання продовжується **після** всіх блоків `catch` поточного `try`.

```cpp
#include <iostream>
using namespace std;

int divide(int a, int b) {
    if (b == 0)
        throw runtime_error("Ділення на нуль!");
    return a / b;
}

int main() {
    try {
        cout << divide(10, 0) << endl;
    }
    catch (runtime_error& e) {
        cout << "Виняток: " << e.what() << endl;
    }
    return 0;
}
```

**Виведення:**
```
Виняток: Ділення на нуль!
```

## 4. Чи можна використовувати `throw` поза блоком `try`?

**Так**, `throw` можна використовувати поза блоком `try`. Однак якщо виняток не буде перехоплено жодним блоком `catch` у стеку викликів, буде викликана стандартна функція `std::terminate()`, яка **аварійно завершує програму**.

```cpp
void foo() {
    throw 42; // виняток кинуто поза try
}

int main() {
    foo(); // виняток не перехоплено → terminate()
}
```

> ⚠️ Якщо `throw` використовується без аргументів **поза** блоком `catch`, це також призводить до виклику `std::terminate()`.

---

## 5. Форма `catch`, що обробляє всі типи винятків

Для перехоплення **будь-якого** типу виняткової ситуації використовується спеціальна форма з трьома крапками:

```cpp
catch (...) {
    // обробляє виняток будь-якого типу
}
```

Цей блок зазвичай розміщують **останнім** серед усіх `catch`-блоків як "запасний" обробник.

```cpp
try {
    throw 3.14; // тип double
}
catch (int e) {
    cout << "int: " << e << endl;
}
catch (...) {
    cout << "Невідомий виняток!" << endl; // спрацює цей
}
```

---

## 6. Що відбудеться, якщо для виняткової ситуації не задано відповідного `catch`?

Якщо виняток не перехоплено жодним відповідним блоком `catch`:

1. C++ **розгортає стек** (stack unwinding) — викликаються деструктори для всіх локальних об'єктів.
2. Викликається стандартна функція **`std::terminate()`**.
3. За замовчуванням `std::terminate()` викликає **`std::abort()`**, що призводить до **аварійного завершення програми**.

```cpp
int main() {
    try {
        throw string("Помилка"); // тип string
    }
    catch (int e) {
        // не перехопить — тип не збігається
    }
    // виняток не оброблено → std::terminate() → abort()
}
```

> 💡 Щоб уникнути аварійного завершення, рекомендується завжди додавати `catch (...)` як останній блок або реєструвати власний обробник через `std::set_terminate()`.

## Підсумкова таблиця

| Оператор | Роль |
|----------|------|
| `try { }` | Визначає блок коду, де можливий виняток |
| `throw вираз;` | Генерує (кидає) виняток |
| `catch (тип e) { }` | Перехоплює виняток конкретного типу |
| `catch (...) { }` | Перехоплює виняток **будь-якого** типу |
| `std::terminate()` | Викликається при необробленому винятку |
